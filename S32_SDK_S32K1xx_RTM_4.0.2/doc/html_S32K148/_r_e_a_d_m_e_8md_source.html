3 ## Note 1
4 ```
5 wolfSSL as of 3.6.6 no longer enables SSLv3 by default. wolfSSL also no
6 longer supports static key cipher suites with PSK, RSA, or ECDH. This means
7 if you plan to use TLS cipher suites you must enable DH (DH is on by default),
8 or enable ECC (ECC is on by default on 64bit systems), or you must enable static
9 key cipher suites with
10  WOLFSSL_STATIC_DH
11  WOLFSSL_STATIC_RSA
12  or
13  WOLFSSL_STATIC_PSK
14 
15 though static key cipher suites are deprecated and will be removed from future
16 versions of TLS. They also lower your security by removing PFS. Since current
17 NTRU suites available do not use ephemeral keys, WOLFSSL_STATIC_RSA needs to be
18 used in order to build with NTRU suites.
19 
20 
21 When compiling ssl.c, wolfSSL will now issue a compiler error if no cipher suites
22 are available. You can remove this error by defining WOLFSSL_ALLOW_NO_SUITES
23 in the event that you desire that, i.e., you're not using TLS cipher suites.
24 ```

26 ## Note 2
27 ```
28 
29 wolfSSL takes a different approach to certificate verification than OpenSSL
30 does. The default policy for the client is to verify the server, this means
31 that if you don't load CAs to verify the server you'll get a connect error,
32 no signer error to confirm failure (-188). If you want to mimic OpenSSL
33 behavior of having SSL_connect succeed even if verifying the server fails and
34 reducing security you can do this by calling:
35 
36 wolfSSL_CTX_set_verify(ctx, SSL_VERIFY_NONE, 0);
37 
38 before calling wolfSSL_new(); Though it's not recommended.
39 ```
